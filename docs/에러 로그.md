# 에러 로그 (Error Log)

이 문서는 개발 중 발생했던 주요 에러와 해결 방법을 기록합니다. 앞으로 발생하는 에러들도 이곳에 누적하여 관리합니다.

## 1. 쿠키 수정 (비동기 환경) 오류
**에러 메시지:**
`Cookies can only be modified in a Server Action or Route Handler. Read more: https://nextjs.org/docs/app/api-reference/functions/cookies#options`
**발생 위치:** `logout` (src/app/actions/auth.ts)
**원인:** Next.js App Router 환경에서 쿠키(`cookies()`)를 수정/삭제하려고 할 때, 이 작업이 명시적인 서버 액션(`'use server'`)이나 라우트 핸들러 내에서 호출되지 않아 발생한 문제입니다.
**해결 내용:** `logout` 함수에 `'use server'` 디렉티브가 정상적으로 적용되고 호출되도록 구조를 수정하여 쿠키 삭제(`cookieStore.delete`)가 서버 액션의 규칙에 따라 처리될 수 있도록 하였습니다.

## 2. 대시보드 무한 새로고침 및 페이지 렌더링 불가 오류 (리다이렉트 루프)
**에러 증상:** 로컬 환경(`localhost`)에서 대시보드 접근 시 화면이 무한 새로고침되거나 백지 상태(빈 화면)가 노출되는 현상.
**원인:** `middleware.ts`에서 언어 설정(Locale)을 처리하기 위한 리다이렉트 로직이 무한 루프를 발생시켰거나 (ex. `/ko` -> `/ko` 연쇄), JPY(엔화) 통화 설정이나 일본어(ja) 경로로 앱이 진입하려고 할 때 올바른 폴백 앵커가 없어서 렌더링에 실패했습니다.
**해결 내용:** 
- `middleware.ts` 내의 i18n Next-Intl 라우팅 규칙을 언어(ko, en)에 대해서만 정확히 처리하도록 수정하고 `ja`를 완전히 제외하였습니다.
- 리다이렉트를 처리할 때 루프에 빠지지 않도록 locale 체크 정규식을 정교화했습니다.
- 일본어(`ja`) 및 엔화(`JPY`) 잔여 코드를 일괄 제거하여 존재하지 않는 참조로 인한 런타임 크래쉬를 방지했습니다.

## 3. 칸반 보드 카드 드래그 앤 드랍 불가 버그
**에러 증상:** 대시보드 하단의 Campaigns 항목에서 카드를 다른 상태로 드래그 앤 드롭하지 못하는 현상.
**원인:** Framer Motion 혹은 `@dnd-kit` 관련 라이브러리 조합 중, 클라이언트 사이드에서 좌표(x, y)를 제대로 인식하지 못하거나 업데이트 핸들러가 누락된 경우.
**해결 내용:** Kanban 컴포넌트(`CampaignKanban.tsx` 등) 내부의 드래그 허용 범위 및 제약조건(`dragConstraints`)을 재설정하고 브라우저 렌더링 이벤트를 지연 로드하여 컴포넌트 마운트 이후에 정상적으로 동작하게 수정하였습니다.
