# 🚀 B2B SaaS 대시보드 미완성 기능 및 향후 개발 로드맵 (Roadmap)

본 보고서는 현재까지 구현된 프로젝트의 상태를 기반으로 **아직 연동되지 않았거나 미완성된 기능들(Missing Features)**을 식별하고, 실제 **한국 시장(South Korea)에서 즉각적으로 상용화하여 사용할 수 있는 마케팅 플랫폼**으로 완성하기 위한 **향후 개발 계획(Future Roadmap)**을 단계별로 제안합니다.

---

## 🚧 1. 현재 미완성 및 누락된 기능 (Missing Features)

코드베이스와 UI 구조를 분석한 결과, 화면(UI) 상에는 존재하지만 실제 동작이 연결되지 않은 '속 빈 강정' 상태의 기능들이 다수 발견되었습니다.

### 🔴 프론트엔드 (Next.js) 미구현 내역
*   **다중 라우팅(Pages) 부재:** 현재 `/app/page.tsx`(대시보드 홈) 단일 페이지만 존재합니다. 사이드바(Sidebar)에 있는 `Campaigns`, `Analytics`, `Settings`, `Team` 등의 다른 메뉴를 클릭해도 이동할 페이지가 없습니다.
*   **인증(Auth) UI 처리 누락:** 로그인/회원가입 페이지(`/login`, `/register`)가 없으며, 현재 헤더의 로그인 사용자 프로필은 정적인 데이터(가짜 데이터)로 렌더링되고 있습니다.
*   **상태 갱신(Refetch) 문제:** `page.tsx`가 Server Component로 구성되어 최초 로딩은 매우 빠르나, 클라이언트에서 새 캠페인을 추가하거나 삭제한 후 전체 대시보드 차트나 숫자를 즉각적으로 다시 갱신(Revalidation)하는 로직이 부족합니다.
*   **모바일 반응형 최적화 부족:** 태블릿/모바일 화면에서 사이드바가 접히거나(Hamburger Menu) 데이터 테이블이 가로로 스크롤되는 모바일 친화적인 처리가 완벽하지 않습니다.

### 🔴 백엔드 (FastAPI) 미구현 내역
*   **Auth 로직 파편화:** `src/domains/auth`에 로그인 서비스, 토큰 발급 로직이 존재하지만 API 라우터에 제대로 연결되지 않았거나, 프론트엔드와 JWT를 주고받는 미들웨어 설계가 미완성입니다.
*   **멀티테넌시(Multi-Tenancy) 하드코딩:** 현재 모든 캠페인 데이터 통신에서 `organization_id=12`라는 값을 프론트엔드(`page.tsx` 및 `route.ts`)에서 하드코딩하여 사용 중입니다. 실제 로그인한 유저의 소속 조직(Organization) ID를 동적으로 받아와야 합니다.
*   **데이터 검증 및 에러 처리:** 캠페인 추가/삭제 시 예외 상황(예: 권한 없음, 데이터 포맷 오류)에 대한 디테일한 HTTP 상태 코드 및 에러 메시지 반환 처리가 부족합니다.

## 🗺️ 2. 향후 개발 로드맵 (Future Roadmap)

위의 미완성 요소들을 해결하고 제품을 완성하기 위한 단계별 실행 계획입니다.

### 🏃 Phase 1: 인증 및 멀티테넌시 기반 공사 (1~2주차)
가장 시급한 것은 하드코딩된 데이터를 걷어내고 실제 유저 기반으로 동작하게 만드는 것입니다.

1.  **백엔드 인증 고도화:** FastAPI의 Auth 도메인을 완성하고 JWT Login 엔드포인트를 확립합니다.
2.  **프론트엔드 로그인 페이지 구현:** `app/login/page.tsx`를 생성하고 Next-Auth(Auth.js) 또는 커스텀 Context를 이용해 세션 관리를 도입합니다.
3.  **동적 Organization ID 적용:** 하드코딩된 `?organization_id=12`를 제거하고, 로그인한 유저의 토큰에서 추출한 실제 조직 ID를 API 요청 시 담아 보내도록 전면 수정합니다.

### 🏃 Phase 2: 서브 페이지 및 상태 관리 연동 (3~4주차)
단일 페이지 애플리케이션의 한계를 벗어나 다중 페이지 구조를 잡습니다.

1.  **사이드바 메뉴 라우팅 연결:** `/campaigns`, `/analytics`, `/settings` 등의 하부 디렉토리와 `page.tsx`를 생성하여 화면을 분할합니다.
2.  **전역 상태 또는 데이터 뮤테이션 적용:** React Query(강력 추천) 또는 Next.js Server Actions(revalidatePath)를 도입하여, 캠페인 목록에서 항복을 삭제/추가했을 때 즉각적으로 서버 데이터를 갱신하고 상단 통계 위젯에 반영되도록 만듭니다.
3.  **차트 데이터 동적 연동:** 현재 정적인 가짜 데이터(Mock)로 그려지는 `PerformanceChart`에 실제 캠페인 클릭/전환수 DB 데이터를 연결합니다.

### 🏃 Phase 3: 고도화 및 안정성 확보 (5주차~)
서비스 런칭 전 UX와 안정성을 챙깁니다.

1.  **UI/UX 폴리싱:** 모바일 환경에서의 사이드바 햄버거 메뉴 및 테이블 스크롤 최적화 등 반응형 웹 디자인(RWD)을 완벽하게 적용합니다.
2.  **에러 핸들링 및 로딩 UI:** API 호출 지연 시 스켈레톤(Skeleton) UI를 보여주고, 실패 시 사용자 친화적인 에러 토스트(Toast/Snackbar) 알림을 글로벌하게 띄우도록 설정(`error.tsx` 및 `loading.tsx` 적극 활용)합니다.
3.  **로깅 및 모니터링 적용:** Sentry 같은 에러 트래킹 툴을 붙여 Cloudflare/Render 환경에서 발생하는 프론트/백엔드 런타임 에러를 실시간으로 수집합니다.

### 🏃 Phase 4: 다크 모드 및 테마 시스템 (2026-03 초순)
본격적인 B2B SaaS의 필수 기능인 다크 모드(Dark Mode)를 완벽하게 지원합니다.

1.  **Tailwind CSS Dark Mode 연동:** `next-themes` 라이브러리를 도입하여 운영체제 환경 및 사용자 선택에 따른 완전한 테마 전환(Light/Dark/System) 기능을 시스템 레벨에 구성합니다.
2.  **컬러 팔레트(Color Palette) 토큰화:** 하드코딩된 `#3b82f6`(블루), `gray-50` 등을 디자인 시스템 토큰(CSS Variables)으로 추출하여 통일성 있는 색상 전환을 이룩합니다.
3.  **다크 모드 전용 차트(Chart) 테마:** Recharts 등 서드파티 라이브러리 차트 구성요소의 선, 그리드, 텍스트 색상도 다크 모드에 맞춰 반전(Invert)되도록 훅(Hook) 단위 처리를 구성합니다.

### 🏃 Phase 5: 애니메이션 및 마이크로 인터랙션 (2026-03 중순)
단순한 화면 전환을 넘어 사용자에게 '프리미엄' 한 체감 성능과 시각적 몰입감을 제공합니다.

1.  **Framer Motion 도입:** Next.js 라우팅 전환(페이지 이동) 시 끊기지 않고 자연스럽게 스며드는 레이아웃 전환(Page Transitions) 애니메이션을 구축합니다.
2.  **위젯(Widget) 마이크로 인터랙션:** 대시보드 진입 시 상단 통계 카드, 차트, 리스트들이 동시에 뜨지 않고 약 `0.1s` 간격으로 부드럽게 튀어오르는 순차 렌더링(Staggered Fade-in) 기법을 적용합니다.
3.  **피드백 애니메이션:** 폼(Form) 제출 성공/실패, 버튼 호버(Hover), 탭(Tab) 메뉴 전환 등 사용자 액션에 대한 시각적 피드백 효과를 추가합니다.

### 🏃 Phase 6: 한국형 맞춤 실무 분석(Analytics) 및 커스텀 대시보드 (2026-03 하순)
정적인 데이터를 넘어, 한국 마케터들이 실제로 자주 보는 지표와 데이터를 교차 분석할 수 있는 환경을 제공합니다.

1.  **매체별 퍼포먼스 비교 차트:** 네이버(Naver), 카카오(Kakao), 메타(Meta), 구글(Google) 등 매체별 캠페인 성과(ROAS, 예산 소진율)를 한눈에 비교하는 Radar 차트 및 Bar 차트를 도입합니다.
2.  **한국 기준 날짜 필터(Date Picker):** '어제', '최근 7일', '이번 달' 등 실무에서 주로 쓰는 기준일 필터링을 도입하고, KST(한국 표준시) 기준으로 데이터가 정확히 집계되도록 백엔드/프론트엔드 연동을 강화합니다.
3.  **조직(Org) 초청 권한 모델:** `/users` 페이지를 완성하여 에이전시(대행사)나 인하우스 스태프 등 이메일로 팀원을 초대하고 'Admin', 'Viewer' 등 역할(Role)에 따른 렌더링 분기(RBAC)를 시스템에 병합합니다.

### 🇰🇷 Phase 7: 진정한 국내 상용화 (Local Integration) (추가 로드맵)
진정한 '한국에서 쓸 수 있는 플랫폼'이 되기 위한 필수 연동 및 UX 고도화를 진행합니다.

1.  **국내 주요 광고 매체 API 연동(추후 백엔드):** 사용자가 직접 숫자를 입력하는 방식에서 벗어나, 네이버 검색광고 API, 카카오 모먼트 API 등을 연동하여 자동으로 캠페인 데이터를 긁어오는 완전 자동화(Automated Sync)의 초석을 다집니다.
2.  **원화(KRW) 및 엑셀(CSV) 추출 완벽 지원:** 대시보드의 모든 테이블 로직을 원화 포맷(₩, 콤마 처리)으로 강제 고정하고, 한글 깨짐 현상(EUC-KR/UTF-8 믹스)이 없는 완벽한 "엑셀 다운로드(Export to CSV/Excel)" 기능을 지원하여 국내 실무자 편의를 극대화합니다.
3.  **카카오 알림톡 / 슬랙 웹훅 지원:** ROAS가 급락하거나 예산이 초과 소진되는 등 'AI Insight' 카드에서 감지되는 위험 신호를 카카오톡이나 슬랙(Slack)으로 관리자에게 즉시 쏴주는 알림 시스템 설계를 고민합니다.

### 🧠 Phase 8: AI 예산 자동 최적화 에이전트 (AI Budget Optimizer)
기존의 단순 모니터링을 넘어, AI가 캠페인의 성과를 분석하고 스스로 행동하는 영역입니다.
1.  **예산/입찰가 자동 추천 로직:** 개별 캠페인의 목표 ROAS를 밑도는 경우, AI가 과거 데이터를 분석해 예산을 축소하거나 입찰가를 조정하도록 추천(혹은 자동 조정) 모델을 삽입합니다.
2.  **A/B 테스트 자동화:** 랜딩 페이지나 광고 소재의 A/B 테스트 결과를 AI가 분석하여, 승리한 소재(Winner) 쪽으로 예산을 자동으로 집중(Auto-shift)시킵니다.
3.  **LLM 기반 마케팅 분석 챗봇:** 캠페인 대시보드 내에 챗봇을 배치하여, "지난주 구글 검색광고에서 예산 소진이 가장 컸던 캠페인이 뭐야?" 등의 자연어 질의(NLQ)를 통한 데이터 인사이트를 제공합니다.

### 💸 Phase 9: B2B 결제 및 구독형 SaaS 전환 (Billing & Subscription)
실제 수익 창출을 위한 B2B 구독 결제 파이프라인과 플랜별 과금 체계를 도입합니다.
1.  **결제 게이트웨이(PortOne, Stripe 등) 연동:** 월간/연간 정기 결제 기능을 연동하고, 자동 인보이스(Invoice) 발행 기능을 시스템화합니다.
2.  **멤버십 플랜 및 데이터 티어링(Tiering):** Basic, Pro, Enterprise 등 티어(Tier)를 나누고, 연동 가능한 광고 매체 수나 매월 AI 분석 횟수 제한 로직을 API 레벨에서 제어합니다.
3.  **조직(Organization) 단위 통합 결제:** 개별 사용자가 아닌, 회사/에이전시 단위로 결제 수단을 등록하고 청구서를 통합 관리할 수 있는 Billing 대시보드를 구축합니다.

### 📊 Phase 10: 커스텀 리포트 생성 및 화이트라벨링 (Custom Reports & White-labeling)
에이전시(광고 대행사)나 비즈니스 중간 관리자를 위해 브랜딩이 가능하고 보고에 특화된 기능을 개발합니다.
1.  **스케줄링 리포트 자동 발송:** 스케줄러(Cron 잡)를 구축하여 매주/매월 캠페인 성과 요약 리포트를 PDF나 데일리 이메일 형태로 클라이언트에게 자동 발송합니다.
2.  **화이트라벨(White-label) 클라이언트 대시보드:** 접속한 클라이언트(광고주)에 따라 대시보드의 로고, 브랜드 컬러, 로그인 도메인(CNAME)을 광고대행사 자체 브랜드로 바꿀 수 있는 기능을 제공합니다.
3.  **동적 리포트 빌더(Report Builder):** 고정된 대시보드가 아닌, 사용자가 드래그 앤 드롭으로 차트와 표를 배치하여 나만의 커스텀 뷰(View)를 생성하고 공유하는 기능을 추가합니다.

### 🎯 Phase 11: 1st Party 데이터 및 CRM 옴니채널 연동 (Audience & CRM Sync)
외부 광고 데이터를 넘어 내부 고객 데이터(User Data) 기반으로 리타겟팅과 페르소나를 고도화합니다.
1.  **마케팅 파이프라인 연동:** 자사몰 회원가입 데이터, CRM(Salesforce, Hubspot 등) 정보를 웹훅/API로 수집하여 광고 매체의 전환(Conversion) 데이터와 결합합니다.
2.  **크로스 채널 관객 동기화(Audience Sync):** 대시보드 내에서 특정 조건(예: 최근 30일 내 고액 결제 유저)에 맞는 유저 풀을 추출한 뒤, 버튼 클릭 하나로 메타(Meta)나 구글의 맞춤 타겟(Custom Audience)으로 즉시 동기화합니다.
3.  **고객 생애 가치(LTV) 추적 코어:** 단기적인 ROAS를 넘어, 유입된 고객이 장기적으로 발생시킨 총 매출(LTV)을 추적하고 시각화하는 모듈을 도입합니다.

### 🏢 Phase 12: 엔터프라이즈급 보안 및 감사 시스템 (Enterprise Security & Compliance)
대형 기업 고객(Enterprise)을 유치하기 위한 엔터프라이즈급 인프라와 강력한 보안을 완성합니다.
1.  **SSO(Single Sign-On) 및 SAML 지원:** Okta, Google Workspace 등 사내 인트라넷 계정으로 강제 로그인하게 만들어 기업 수준의 인증 보안을 확립합니다.
2.  **불변 감사 로그(Immutable Audit Logs) 페이지:** '누가, 언제, 어떤 캠페인 예산을 수정하고 중지시켰는지(Action/IP/Time)' 완벽하게 전부 기록하여 책임 소재를 명확히 하는 시스템을 구현합니다.
3.  **ISMS / SOC2 대응 인프라 격리:** 고객사(Organization)별로 데이터베이스 테넌트(Tenant/Schema)를 물리적으로 분리하거나, 민감 개인정보(PII)를 철저하게 양방향 암호화하여 저장소에 보관하는 규제(Compliance) 대응 아키텍처를 세팅합니다.

### 🔐 Phase 13: 런칭 직전 필수 디테일 주입 (Missing Links 여백 채우기)
지금 당장 고객에게 상용 오픈(Open Beta)하기 위해 마무리되지 않은 마지막 '실수 방지망'과 연결 고리를 100% 채웁니다.
1. **RBAC (역할 기반 접근 제어) 마감:** 프론트엔드 UI(`/users`)에만 존재하는 Admin, Editor, Viewer 뱃지를 실제 백엔드 API 권한 검증(JWT Payload) 미들웨어와 연결하여, Viewer 권한자가 실수로 캠페인 삭제 등을 시도할 때 `403 Forbidden` 처리되도록 권한 분리.
2. **동적 세션(Session) 100% 전파:** 대시보드 곳곳이나 최근 추가된 Audit Log 시스템 등에서 아직 임시 하드코딩 되어있는 `organization_id=12` 스니펫들을 색출하여, 실제 Next-Auth(세션) 혹은 JWT 토큰 파싱 값으로 전부 동적 대체.
3. **서드파티 Webhook 실반영 (Slack/Kakao):** `/settings`에서 입력받은 알림톡/슬랙 Webhook URL을 백엔드 DB와 연동하여, 목표 ROAS 미달이나 예산 한도 초과 등 치명적인 이슈 발동 시 파이썬 백엔드 스케줄러가 실제로 메시지를 쏴주도록 트리거 완전 개통.

### 🤝 Phase 14: 워크플로우 자동화 및 결재 시스템 (Workflow & Approvals)
규모가 큰 에이전시나 기업 실무에서 필수적인 '협업'과 '결재' 라인을 대시보드 내에 구축합니다.
1.  **결재 프로세스(Approval Lines) 도입:** 마케터가 대규모 예산을 증액하거나 새로운 캠페인을 라이브하기 전에, 팀장이나 광고주의 승인(Approve/Reject)을 거치도록 강제하는 워크플로우를 추가합니다.
2.  **칸반 보드(Kanban) 뷰:** 캠페인 상태(기획 중, 결재 대기 중, 라이브, 종료)를 칸반 보드 형태로 시각화하여 팀원 간 업무 진행 상황을 직관적으로 공유합니다.
3.  **내부 코멘트 및 멘션(@) 시스템:** 개별 캠페인 상세 페이지에 팀원끼리 의견을 나눌 수 있는 쓰레드 형태의 댓글(Comment) 기능과 이메일 푸시가 연동되는 태그(@) 기능을 구현합니다.

### 🔮 Phase 15: 수요 예측 및 지표 기반 타겟 분석 (Predictive Analytics)
과거 데이터를 보여주는 것을 넘어, 머신러닝 기반 시계열 예측(Time Series Forecasting)으로 미래 지표를 선제적으로 파악합니다.
1.  **ROAS 및 예산 소진 예측 차트:** 현재 추세를 기반으로 월말 예상 광고 지출액과 전환 수를 계산하여 보여주는 트렌드 캐스팅(Trendcasting) 위젯을 개발합니다.
2.  **이상 탐지 알림(Anomaly Detection):** 평소 패턴과 다른 급격한 클릭수 하락이나 예산 급증 현상을 탐지 모델(Isolation Forest 등)로 파악하여 사전 알림을 발송합니다.
3.  **계절성(Seasonality) 분석 및 대비:** 특정 시즌이나 요일별 효율 편차를 분석해, 명절 전후 등 특정 주기에 입찰가를 올리도록 제안하는 캘린더 기능을 도입합니다.

### 📱 Phase 16: 모바일 네이티브 대응 및 PWA (Mobile App & Push)
언제 어디서나 캠페인 비상 징후를 모니터링할 수 있도록 완벽한 모바일 환경을 구축합니다.
1.  **PWA(Progressive Web App) 아키텍처:** 데스크톱 웹을 넘어, 모바일 기기의 홈 화면에 앱처럼 설치하고 캐시를 통해 오프라인 환경에서도 일부 열람이 가능하도록 설정합니다.
2.  **모바일 네이티브 푸시(Native Push):** Firebase Cloud Messaging(FCM) 통신망을 연동하여 데스크톱 브라우저가 꺼져 있어도 긴급 알림(예산 급격 소진 등)을 스마트폰 모바일 푸시 알림으로 전달합니다.
3.  **모바일 최적화 핸드 헬드 인터랙션:** 터치 화면에서 캠페인 리스트를 좌우로 스와이프하여 일시정지하거나 삭제할 수 있는 네이티브 앱 수준의 손맛(UX) 제스처를 적용합니다.

### 🌍 Phase 17: 글로벌 런칭을 위한 다국어 및 로컬 현지화 (Global Expansion)
국내 시장 상용화를 넘어, 북미/일본 등 해외 글로벌 고객 유치를 위한 i18n 환경을 조성합니다.
1.  **다국어(i18n) 번역 파이프라인 구성:** `next-intl` 또는 `react-i18next`를 도입하여 영어(EN), 일본어(JP) 등 사용자 언어 설정에 따라 대시보드 텍스트가 실시간 핫스와핑 되도록 파이프라인을 구축합니다.
2.  **다중 통화(Multi-Currency) 및 실시간 환율 동기화:** KRW뿐만 아니라 USD, JPY 등 다양한 통화로 실적을 볼 수 있게 통화 스위처(Currency Switcher)를 도입하고, 실시간 환율 API(OpenExchangeRates 등)와 연동합니다.
3.  **글로벌 결제 수단 및 조세(Tax) 처리 분기:** Stripe 기반의 해외 결제 시 국가별 부가세(VAT, GST)를 자동 계산하고 영수증 포맷을 국가 표준 세법에 맞추는 글로벌 엔터프라이즈 빌링 시스템을 마무리합니다.

### 🤝 Phase 18: B2B CRM 연동 및 마케팅 자동화 (CRM & Marketing Automation)
엔터프라이즈 B2B 고객을 위한 외부 리드(Lead) 연동과 이메일 자동화 파이프라인.
1. **Salesforce / HubSpot OAuth 연동:** 써드파티 인그레이션을 통해 캠페인으로 유입된 고객 리드를 양방향 전송.
2. **이메일 드립 캠페인(Drip Campaign) 빌더:** 대시보드에서 직접 이메일 템플릿을 드래그 앤 드롭으로 구성하고, 단계별로 자동 메일(Welcome, Follow-up)을 발송하는 워크플로우 엔진.

### 🎬 Phase 19: AI 광고 소재 및 카피 무한 생성기 (AI Creative Generation)
텍스트 분석을 뛰어넘는 Generative AI 기반 콘텐츠 자동 양산기.
1. **DALL-E / Midjourney 배너 생성기:** 캠페인 주제 입력 시 광고 배너용 배경 이미지와 문구가 들어간 크리에이티브 수십 장을 5초 만에 생성.
2. **다국어 A/B 테스트용 카피라이팅 엔진:** GPT-4를 활용하여 한 언어로 작성한 카피를 타겟 시장의 뉘앙스에 맞는 10개국 언어별 A/B 카피로 변형.

### 🔐 Phase 20: 오프라인/O2O 성과 옴니채널 연동 (Omnichannel Attribution)
온라인에 국한되지 않은 오프라인 마케팅 기여도(Attribution) 분석.
1. **동적 QR 코드 생성 및 트래킹:** 옥외광고나 지류 쿠폰용 동적 QR을 생성하여 오프라인 유입 성과를 대시보드 트래픽과 결합.
2. **매장 POS 데이터 동기화:** 오프라인 결제 데이터를 API로 연동(Sync)받아, 어떤 온라인 캠페인이 실제 오프라인 매장 매출 지표로 환산되었는지 옴니채널 LTV/ROAS 연산.

### 🦄 Phase 21: 엔터프라이즈 데이터 웨어하우스 분양 (Data Warehouse Export)
데이터 주도권(Sovereignty)을 요구하는 대기업용 전용 인프라 연동망.
1. **Bring Your Own DB (BYODB):** 클라이언트가 사용하는 자체 클라우드(AWS S3, Snowflake, BigQuery)로 대시보드 데이터를 매일 자정에 자동 덤프(Dump) 지원.
2. **실시간 트래픽 Firehose 연동:** Amazon Kinesis 또는 Kafka 커넥터를 제공하여, 캠페인 클릭/전환 원천 로그 스트림을 딜레이 없이 고객사 데이터팀에 제공.

### 🪐 Phase 22: 마켓플레이스 및 써드파티 플러그인 생태계 (Platform Marketplace)
독립형 SaaS를 넘어, 외부 앱들이 연동되는 에코시스템/플랫폼으로의 확장.
1. **Developer Open API & Webhooks:** 외부 개발자가 대시보드 캠페인을 제어하거나 데이터를 추출할 수 있는 완전체 Open API 콘솔 발행.
2. **플러그인 앱 스토어 (App Store):** 외부 업체가 만든 '디스코드 알림 봇', '틱톡 광고 커넥터' 등을 사용자가 클릭 하나로 대시보드에 설치(Install)할 수 있는 플러그인 생태계 구축.
