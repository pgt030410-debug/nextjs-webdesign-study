# 🚀 B2B SaaS 대시보드 미완성 기능 및 향후 개발 로드맵 (Roadmap)

본 보고서는 현재까지 구현된 프로젝트의 상태를 기반으로 **아직 연동되지 않았거나 미완성된 기능들(Missing Features)**을 식별하고, 이를 완성하여 상용 서비스 수준으로 끌어올리기 위한 **향후 개발 계획(Future Roadmap)**을 단계별로 제안합니다.

---

## 🚧 1. 현재 미완성 및 누락된 기능 (Missing Features)

코드베이스와 UI 구조를 분석한 결과, 화면(UI) 상에는 존재하지만 실제 동작이 연결되지 않은 '속 빈 강정' 상태의 기능들이 다수 발견되었습니다.

### 🔴 프론트엔드 (Next.js) 미구현 내역
*   **다중 라우팅(Pages) 부재:** 현재 `/app/page.tsx`(대시보드 홈) 단일 페이지만 존재합니다. 사이드바(Sidebar)에 있는 `Campaigns`, `Analytics`, `Settings`, `Team` 등의 다른 메뉴를 클릭해도 이동할 페이지가 없습니다.
*   **인증(Auth) UI 처리 누락:** 로그인/회원가입 페이지(`/login`, `/register`)가 없으며, 현재 헤더의 로그인 사용자 프로필은 정적인 데이터(가짜 데이터)로 렌더링되고 있습니다.
*   **상태 갱신(Refetch) 문제:** `page.tsx`가 Server Component로 구성되어 최초 로딩은 매우 빠르나, 클라이언트에서 새 캠페인을 추가하거나 삭제한 후 전체 대시보드 차트나 숫자를 즉각적으로 다시 갱신(Revalidation)하는 로직이 부족합니다.
*   **모바일 반응형 최적화 부족:** 태블릿/모바일 화면에서 사이드바가 접히거나(Hamburger Menu) 데이터 테이블이 가로로 스크롤되는 모바일 친화적인 처리가 완벽하지 않습니다.

### 🔴 백엔드 (FastAPI) 미구현 내역
*   **Auth 로직 파편화:** `src/domains/auth`에 로그인 서비스, 토큰 발급 로직이 존재하지만 API 라우터에 제대로 연결되지 않았거나, 프론트엔드와 JWT를 주고받는 미들웨어 설계가 미완성입니다.
*   **멀티테넌시(Multi-Tenancy) 하드코딩:** 현재 모든 캠페인 데이터 통신에서 `organization_id=12`라는 값을 프론트엔드(`page.tsx` 및 `route.ts`)에서 하드코딩하여 사용 중입니다. 실제 로그인한 유저의 소속 조직(Organization) ID를 동적으로 받아와야 합니다.
*   **데이터 검증 및 에러 처리:** 캠페인 추가/삭제 시 예외 상황(예: 권한 없음, 데이터 포맷 오류)에 대한 디테일한 HTTP 상태 코드 및 에러 메시지 반환 처리가 부족합니다.

## 🗺️ 2. 향후 개발 로드맵 (Future Roadmap)

위의 미완성 요소들을 해결하고 제품을 완성하기 위한 단계별 실행 계획입니다.

### 🏃 Phase 1: 인증 및 멀티테넌시 기반 공사 (1~2주차)
가장 시급한 것은 하드코딩된 데이터를 걷어내고 실제 유저 기반으로 동작하게 만드는 것입니다.

1.  **백엔드 인증 고도화:** FastAPI의 Auth 도메인을 완성하고 JWT Login 엔드포인트를 확립합니다.
2.  **프론트엔드 로그인 페이지 구현:** `app/login/page.tsx`를 생성하고 Next-Auth(Auth.js) 또는 커스텀 Context를 이용해 세션 관리를 도입합니다.
3.  **동적 Organization ID 적용:** 하드코딩된 `?organization_id=12`를 제거하고, 로그인한 유저의 토큰에서 추출한 실제 조직 ID를 API 요청 시 담아 보내도록 전면 수정합니다.

### 🏃 Phase 2: 서브 페이지 및 상태 관리 연동 (3~4주차)
단일 페이지 애플리케이션의 한계를 벗어나 다중 페이지 구조를 잡습니다.

1.  **사이드바 메뉴 라우팅 연결:** `/campaigns`, `/analytics`, `/settings` 등의 하부 디렉토리와 `page.tsx`를 생성하여 화면을 분할합니다.
2.  **전역 상태 또는 데이터 뮤테이션 적용:** React Query(강력 추천) 또는 Next.js Server Actions(revalidatePath)를 도입하여, 캠페인 목록에서 항복을 삭제/추가했을 때 즉각적으로 서버 데이터를 갱신하고 상단 통계 위젯에 반영되도록 만듭니다.
3.  **차트 데이터 동적 연동:** 현재 정적인 가짜 데이터(Mock)로 그려지는 `PerformanceChart`에 실제 캠페인 클릭/전환수 DB 데이터를 연결합니다.

### 🏃 Phase 3: 고도화 및 안정성 확보 (5주차~)
서비스 런칭 전 UX와 안정성을 챙깁니다.

1.  **UI/UX 폴리싱:** 모바일 환경에서의 사이드바 햄버거 메뉴 및 테이블 스크롤 최적화 등 반응형 웹 디자인(RWD)을 완벽하게 적용합니다.
2.  **에러 핸들링 및 로딩 UI:** API 호출 지연 시 스켈레톤(Skeleton) UI를 보여주고, 실패 시 사용자 친화적인 에러 토스트(Toast/Snackbar) 알림을 글로벌하게 띄우도록 설정(`error.tsx` 및 `loading.tsx` 적극 활용)합니다.
3.  **로깅 및 모니터링 적용:** Sentry 같은 에러 트래킹 툴을 붙여 Cloudflare/Render 환경에서 발생하는 프론트/백엔드 런타임 에러를 실시간으로 수집합니다.

---

### 🏃 Phase 4: 다크 모드 및 테마 시스템 (2026-03 초순)
본격적인 B2B SaaS의 필수 기능인 다크 모드(Dark Mode)를 완벽하게 지원합니다.

1.  **Tailwind CSS Dark Mode 연동:** `next-themes` 라이브러리를 도입하여 운영체제 환경 및 사용자 선택에 따른 완전한 테마 전환(Light/Dark/System) 기능을 시스템 레벨에 구성합니다.
2.  **컬러 팔레트(Color Palette) 토큰화:** 하드코딩된 `#3b82f6`(블루), `gray-50` 등을 디자인 시스템 토큰(CSS Variables)으로 추출하여 통일성 있는 색상 전환을 이룩합니다.
3.  **다크 모드 전용 차트(Chart) 테마:** Recharts 등 서드파티 라이브러리 차트 구성요소의 선, 그리드, 텍스트 색상도 다크 모드에 맞춰 반전(Invert)되도록 훅(Hook) 단위 처리를 구성합니다.

### 🏃 Phase 5: 애니메이션 및 마이크로 인터랙션 (2026-03 중순)
단순한 화면 전환을 넘어 사용자에게 '프리미엄' 한 체감 성능과 시각적 몰입감을 제공합니다.

1.  **Framer Motion 도입:** Next.js 라우팅 전환(페이지 이동) 시 끊기지 않고 자연스럽게 스며드는 레이아웃 전환(Page Transitions) 애니메이션을 구축합니다.
2.  **위젯(Widget) 마이크로 인터랙션:** 대시보드 진입 시 상단 통계 카드, 차트, 리스트들이 동시에 뜨지 않고 약 `0.1s` 간격으로 부드럽게 튀어오르는 순차 렌더링(Staggered Fade-in) 기법을 적용합니다.
3.  **피드백 애니메이션:** 폼(Form) 제출 성공/실패, 버튼 호버(Hover), 탭(Tab) 메뉴 전환 등 사용자 액션에 대한 시각적 피드백 효과를 추가합니다.

### 🏃 Phase 6: 고급 분석(Analytics) 및 커스텀 대시보드 구성 (2026-03 하순)
정적인 데이터를 넘어 사용자가 직접 원하는 데이터를 비틀어보고 분석할 수 있는 고급 기능을 해금합니다.

1.  **상세 Analytics 차트 고도화:** Bar, Pie, Radar Chart 등을 추가하여 플랫폼별, 광고주별 전환(Conversion) 효율을 입체적으로 교차 분석할 수 있는 화면을 엽니다.
2.  **날짜 필터(Date Picker) 연동:** '최근 7일', '최근 30일', '사용자 지정 기간' 등 날짜 필터링에 따라 서버 액션(Server Actions)을 재호출하여 차트와 테이블이 다이나믹하게 리로드되도록 구현합니다.
3.  **조직(Org) 초청 권한 모델:** `/users` 페이지를 완성하여 실제 이메일로 팀원을 초대하고 'Admin', 'Viewer' 등 역할(Role)에 따른 렌더링 분기(RBAC)를 시스템에 병합합니다.
