# Appearance 고도화 방안 (Appearance Advance)

현재 구현된 Appearance(화이트라벨링) 설정은 B2B SaaS의 기본적인 뼈대(미리 정의된 테마 컬러 선택 + 텍스트 형태의 로고 변경)를 갖추고 있습니다. 이를 '진짜 상용화 수준(Enterprise-grade)의 B2B White-labeling'으로 발전시키기 위한 구체적인 방법론입니다.

---

### 🟢 Level 1: UI/UX 스펙업 (즉시 적용하기 좋은 기능)

**1. 실제 이미지 로고(Logo Upload) 지원**
*   **현재**: 텍스트(예: "SaaS Admin")만 입력받아 상단 그라데이션 텍스트로 보여주고 있습니다.
*   **발전 방향**: 이미지(PNG, SVG)를 업로드받아 사이드바 상단, 로그인 화면, 리포트 표지 등에 박을 수 있도록 만듭니다. `useThemeStore`에 이미지 URL(Base64 또는 AWS S3 링크)을 저장하도록 확장하면 됩니다.

**2. 로그인/회원가입 화면 연동 (Login Page Override)**
*   **현재**: Appearance 변경사항이 로그인 후 대시보드 안에서만 보입니다.
*   **발전 방향**: 유저가 자신들의 고객사(Client) 아이디로 접속할 때, 가장 처음 마주하는 `/login` 화면부터 커스텀 로고와 브랜드 컬러가 입혀진 화면을 보여주어 "우리 회사가 만든 솔루션"이라는 소속감을 줍니다.

**3. 타이포그래피(Font Family) 선택권 부여**
*   **발전 방향**: 현재는 `Inter` 폰트 하나만 사용 중인데, 드롭다운을 통해 '친근한 둥근 폰트(Nunito)', '각진 테크 폰트(Roboto)', 혹은 '세리프 고급 폰트' 등 3~4가지 옵션을 제공하여 브랜드 성격에 맞게 시스템 폰트를 바꿀 수 있게 합니다.

---

### 🟡 Level 2: 커스텀 컬러 고도화 (기술적 챌린지)

**4. 자유로운 HEX/RGB 컬러 피커 (Custom Color Input)**
*   **현재**: 안전성을 위해 Tailwind의 7가지 정해진 색상 셋(Blue, Rose, Emerald 등) 버튼만 선택 가능합니다.
*   **발전 방향**: 사용자가 자기 회사의 정확한 브랜드 컬러(#FF5733 등)를 Hex 코드로 입력하게 만듭니다. 
*   **기술적 해결**: 입력받은 Hex 코드를 기반으로 HSL이나 RGB 값으로 변환하여, Next.js의 최상위 `layout.tsx` 또는 `<body>` 태그에 CSS 변수(`--primary: 15 80% 50%;`) 형태로 동적 주입(Inject)하면 완벽한 무제한 컬러 커스텀이 가능해집니다.

**5. 다크모드/라이트모드 로고 별도 설정**
*   **발전 방향**: 검은색 바탕의 로고 이미지 하나만 업로드하면 다크모드일 때 로고가 안 보일 수 있습니다. 'Light Mode Logo'와 'Dark Mode Logo'를 분리해서 업로드받는 세심한 디테일을 추가합니다.

---

### 🔴 Level 3: 진정한 B2B 엔터프라이즈 기능 (서버 연동 필수)

**6. 커스텀 도메인 (Custom CNAME Routing)**
*   **발전 방향**: 가장 끝판왕 기능입니다. 고객이 `app.saas-admin.com`으로 접속하는 게 아니라, `dashboard.client-company.com`처럼 **자신의 회사 도메인을 연결**하게 해줍니다. Next.js의 Middleware를 사용해 호스트 도메인을 판별하고, 해당 도메인에 맞는 로고와 테마 컬러를 서버에서 내려주도록 설계합니다.

**7. 이메일 템플릿 화이트라벨링**
*   **발전 방향**: 고객상담(CS), 사용자 초대 이메일, 자동화 리포트 PDF 등이 이메일로 날아갈 때도 발신자 이름과 이메일 내부 디자인(버튼 색상 등)을 Appearance 탭에서 설정한 테마에 맞춰 동적으로 생성(SendGrid API 연동 등)해 발송합니다.
